Google Drive Sync — Setup Guide

Choose the method that matches your Google account type

👤

Works with any personal Gmail account

No IT admin needed. Uses OAuth 2.0 — opens a browser sign-in on first run, then saves a token automatically.

Setup Steps

1

Create a free Google Cloud project

Go to console.cloud.google.com → New Project (e.g. project-velocity)

2

Enable Google Drive API

APIs & Services → Library → search "Google Drive API" → Enable

3

Create OAuth 2.0 credentials

APIs & Services → Credentials → Create Credentials → OAuth 2.0 Client ID

Application type: Desktop App → Download JSON

4

Configure OAuth consent screen

APIs & Services → OAuth consent screen → External → Add your Gmail as a Test User

5

Rename & upload the downloaded JSON

Rename to credentials.json and upload it in the Drive Sync card below

6

Set your Drive Folder ID & run sync

On first sync a browser window opens for Google sign-in. After approving, token.json is saved — fully automatic from then on.

🏢

Requires Google Workspace or Google Cloud org access

Uses a Service Account — no browser sign-in needed. Best for automated / server deployments. Contact your IT admin if you don't have Cloud Console access.

Setup Steps

1

Open Google Cloud Console

Go to console.cloud.google.com → select or create a project

2

Enable Google Drive API

APIs & Services → Library → search "Google Drive API" → Enable

3

Create a Service Account

APIs & Services → Credentials → Create Credentials → Service Account

Give it a name (e.g. velocity-drive-sync) → Done

4

Download the JSON key

Click the service account → Keys tab → Add Key → Create new key → JSON → Download

5

Share your Drive folder with the service account

Open the target folder in Drive → Share → paste the client_email from the JSON → set role to Editor

6

Rename & upload the JSON key

Rename to credentials.json and upload it below. No browser sign-in needed — syncs silently.

Both methods use the same credentials.json filename — the app auto-detects which type it is.
