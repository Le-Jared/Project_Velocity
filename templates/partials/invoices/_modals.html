Google Drive Sync — Setup Guide

Choose OAuth for local use or Service Account for automation

👤

Best for local use with Gmail or Workspace user accounts

Uses OAuth 2.0. First sync opens a browser sign-in, then saves token.json for future syncs.

Setup Steps

1

Create a Google Cloud project

Go to console.cloud.google.com → New Project

2

Enable Google Drive API

APIs & Services → Library → search Google Drive API → Enable

3

Configure OAuth consent screen

APIs & Services → OAuth consent screen → choose External or Internal → add your account as a test user if required

4

Create OAuth Desktop credentials

APIs & Services → Credentials → Create Credentials → OAuth client ID → Desktop App → Download JSON

5

Upload the JSON file

Rename it to credentials.json and upload it in the Drive Sync card

6

Set Folder ID and run sync

On first sync, sign in and approve access. Future syncs reuse token.json.

🔐

Best for automated or server deployments

Uses a Service Account key. No browser sign-in needed. The target Drive folder must be shared with the service account email.

Setup Steps

1

Create or open a Google Cloud project

Go to console.cloud.google.com → select or create a project

2

Enable Google Drive API

APIs & Services → Library → search Google Drive API → Enable

3

Create a Service Account

APIs & Services → Credentials → Create Credentials → Service Account

4

Download JSON key

Open the service account → Keys → Add Key → Create new key → JSON

5

Share the target Drive folder

Open the downloaded JSON → copy client_email → share your target Drive folder with that email as Editor

6

Upload the JSON key

Rename it to credentials.json and upload it in the Drive Sync card.

Both methods use credentials.json. The app auto-detects OAuth vs Service Account.
